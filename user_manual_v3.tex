\section{Overview}
% =============================================================

Excel Transfer Tool is a high-performance Qt/C++ desktop application that automates monthly cost-control reporting workflows. It loads Excel workbooks (XLSM/XLSX) for multiple periods, applies configurable transfer mappings, consolidates data across months, and produces clean output files---all without manual copy-paste work.

\begin{infobox}[Key Capabilities --- v1.0]
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item Load multiple cost-control files with parallel processing (2-3x faster)
\item Apply row-level transfer mappings per month/sheet/column
\item Rolling Transfer (RT) mode for sequential month-to-month workflows
\item Parallel Transfer --- run Individual or Rolling modes from a unified backend
\item Fill All Months --- backfill Jan through target month in one operation
\item Ignore Rows --- skip specific rows per mapping card
\item Create Month File --- generate new month files from existing templates
\item Visual Cell Selector --- pick source/destination cells from a grid view
\item Log Panel toggle --- show/hide the status log on demand
\item Quarter quick-select buttons (Q1--Q4) for rapid period selection
\item Automatic formula preservation and shared formula handling
\end{itemize}
\end{infobox}

\subsubsection{How to Use the Main Window}

\begin{stepbox}
\textbf{Step 1: Understanding the Layout}
\begin{enumerate}[leftmargin=*, itemsep=4pt]

\item The \textbf{header toolbar} contains Execute All, Pause, Stop, Reset, and Log buttons.
\item The \textbf{bottom status bar} shows the current operation status and file counts.
\end{enumerate}

\textbf{Step 2: First-Time Setup}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item Verify the application is pointing to the correct network drive (default: L: drive).
\item Ensure all JSON mapping files exist in the \texttt{JSON/} subfolder next to the executable.
\item Check that cost-control files follow the naming convention: \texttt{Cost Control ZAG MM\_YYYY\_working.xlsm}.
\end{enumerate}
\end{stepbox}

\begin{warningbox}[Before You Start]
\textbf{Always close all Excel files} before loading them in the application. Open files will cause ``ZIP read error'' or ``file is locked'' messages and prevent transfers from running.
\end{warningbox}

\subsubsection{Understanding the Toolbar Buttons}

\begin{stepbox}
\textbf{Execute All (Green Button)}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{When to use}: After loading files and checking the mapping cards you want to transfer.
\item \textbf{What it does}: Processes all checked mapping cards in sequence, writing data from source files to destination files.
\item \textbf{Watch out}: Maximum 5 months can be selected at once. More than 5 will show a warning.
\end{itemize}

\textbf{Pause (White Button)}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{When to use}: During a transfer, if you need to temporarily halt execution.
\item \textbf{What it does}: Pauses after the current cell write completes. Click again to resume.
\item \textbf{Watch out}: The current mapping will finish before pausing --- it doesn't stop mid-cell.
\end{itemize}

\textbf{Stop (Red Button)}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{When to use}: To cancel a transfer operation immediately.
\item \textbf{What it does}: Terminates execution. The current sheet finishes saving first to prevent corruption.
\item \textbf{Watch out}: Partial data may be written. Check the log to see which mappings completed.
\end{itemize}

\textbf{Reset (Dark Button)}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{When to use}: To clear all loaded files and start fresh.
\item \textbf{What it does}: Unloads all workbooks, clears mapping cards, and resets period selections.
\item \textbf{Watch out}: This does NOT delete files --- it only clears the application's memory.
\end{itemize}

\textbf{Log (Dark Button)}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{When to use}: To view detailed transfer logs and debug information.
\item \textbf{What it does}: Shows/hides the log panel at the bottom of the window.
\item \textbf{Watch out}: The log panel is hidden by default. Click to show it when troubleshooting.
\end{itemize}
\end{stepbox}

\subsubsection{How to Select Months}

\begin{stepbox}
\textbf{Step 1: Expand a Year Card}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item Click the \textbf{year checkbox} (e.g., ``2025'') to expand the card.
\item The card opens to show 12 month checkboxes (January through December).
\item Quick-select buttons appear on the right: Q1, Q2, Q3, Q4, All, None.
\end{enumerate}

\textbf{Step 2: Select Individual Months}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item \textbf{Click individual month checkboxes} to select specific months (e.g., July, August, September).
\item Selected months turn \textbf{blue} with a checkmark.
\item You can select months from multiple years by expanding multiple year cards.
\end{enumerate}

\textbf{Step 3: Use Quick-Select Buttons}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item \textbf{Q1}: Selects January, February, March
\item \textbf{Q2}: Selects April, May, June
\item \textbf{Q3}: Selects July, August, September
\item \textbf{Q4}: Selects October, November, December
\item \textbf{All}: Selects all 12 months
\item \textbf{None}: Deselects all months in this year
\end{itemize}

\textbf{Step 4: Load the Selected Periods}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item After selecting months, click \textbf{LOAD Selected Periods} at the bottom.
\item Wait for the green ``Loaded'' toast notification.
\item Mapping cards appear in the left sidebar for each loaded month.
\end{enumerate}
\end{stepbox}

\begin{warningbox}[Selection Limit]
\textbf{Maximum 5 months} can be selected for Execute All. Selecting more will show a warning: ``You have selected X months. Maximum is 5 at a time.'' Process months in batches to avoid memory overflow.
\end{warningbox}

\subsubsection{Working with Multiple Years}

\begin{stepbox}
\textbf{When to Select Multiple Years}
\begin{itemize}[leftmargin=*, itemsep=2pt]
\item When you need to process December 2025 and January 2026 together.
\item When backfilling data across a year boundary.
\item When comparing or consolidating data from different fiscal years.
\end{itemize}

\textbf{How to Select Across Years}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item Expand the first year card (e.g., 2025) and select months (e.g., November, December).
\item Expand the second year card (e.g., 2026) and select months (e.g., January, February).
\item The status bar shows total selected: ``4 periods selected''.
\item Click \textbf{LOAD Selected Periods} to load all months from both years.
\end{enumerate}

\textbf{What Happens During Load}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
\item The application scans for cost-control files in each month's folder.
\item For each found file, it loads the workbook structure (sheets, columns, rows).
\item Mapping cards appear in the left sidebar --- one set per month.
\item A green toast notification confirms: ``Loaded 4 period(s)''.
\end{enumerate}

\section{Parallel Transfer (Backend)}
% =============================================================

\subsection{Overview}

The Parallel Transfer system is a new backend that provides two independent transfer modes, each running on its own thread with its own \texttt{ExcelHandler} and \texttt{TransferService} instances:

\begin{center}
\rowcolors{2}{SoftGray}{White}
\begin{tabularx}{0.88\linewidth}{@{} l X @{}}
\toprule
\rowcolor{NewFeature}
\textcolor{White}{\textbf{Mode}} & \textcolor{White}{\textbf{Behaviour}} \\
\midrule
\textbf{Individual} & Each month runs independently. On error: logs and continues to next month. No save+reload between months. \\
\textbf{Rolling}    & Months run sequentially in calendar order. On error: aborts remaining months (data integrity). Save+reload after each month so next month reads fresh output. \\
\bottomrule
\end{tabularx}
\end{center}

\subsection{Key Design Principles}

\begin{itemize}[leftmargin=*, itemsep=4pt]
\item \textbf{Thread safety} --- each worker owns its own handler/service, never shared
\item \textbf{Decoupled} --- deleting the parallel transfer files does not affect existing functionality
\item \textbf{Validation} --- Rolling requires $\geq$2 contiguous months before thread is created
\item \textbf{Cancellable} --- both workers check \texttt{m\_cancelled} flag between entries
\end{itemize}

\subsection{Rolling Validation Rules}

The Rolling worker validates before starting:

\begin{itemize}[leftmargin=*, itemsep=4pt]
\item At least \textbf{2 months} must be selected
\item Months must be \textbf{contiguous} (no gaps, e.g.\ Jan + Mar without Feb is rejected)
\item Validation runs on the \textbf{UI thread} (instant feedback) and again on the \textbf{worker thread} (safety)
\end{itemize}

\section{Troubleshooting}
% =============================================================

\subsection{``not a zip file'' Error}
\begin{warningbox}[Symptom]
\texttt{QZip: not a zip file!} in log. All 9 mappings fail with 0 cells.
\end{warningbox}
\textbf{Cause:} The cost-control xlsm file is corrupt or was created from a corrupted template.\\
\textbf{Fix:} Replace the file with a fresh copy from the Backup folder or copy from another month's working file.

\subsection{0 Cells Transferred}
\begin{itemize}[leftmargin=*, itemsep=4pt]
\item Verify destination file is in the \texttt{Month/} subfolder
\item Check source files (SAP, PAX, Staff) are present and correctly named
\item Ensure no rows are accidentally marked as ``Ignore'' in the Ignore Rows dialog
\item Review log panel for ``sheet not found'' or ``file missing'' warnings
\end{itemize}

\subsection{Stack Overflow / Crash on Execute All}
\begin{warningbox}[Symptom]
Application crashes with exit code 0xC0000602 when clicking Execute All.
\end{warningbox}
\textbf{Cause:} Too many months selected simultaneously creates too many Qt widgets.\\
\textbf{Fix:} Select \textbf{maximum 5 months} at a time. The app will show a warning if more are selected.

\subsection{Fill All Months Writes to Wrong Folder}
Ensure the destination path shown in the scan table ends with \texttt{/Month/}. If it points to the root month folder, the file naming in that folder may be non-standard.

\subsection{Parallel Transfer Not Visible in UI}
The Parallel Transfer backend is implemented but the UI tab is under development. The backend can be connected by wiring \texttt{ParallelTransferController} signals in \texttt{mainwindow.cpp}.

\subsection{Hybrid Transfer Tab}
% ─────────────────────────────────────────────────────────────────────────────

The \textbf{Hybrid Transfer} tab lets you assign individual months to either \textit{Execute All} (cell-by-cell mapping) or \textit{Execute RT} (rolling transfer), then run both modes automatically in sequence. This is useful when some months need a standard transfer while others require a rolling transfer chain.

\textbf{How to use Hybrid Transfer:}
\begin{enumerate}[leftmargin=*, itemsep=4pt]
  \item Select the \textbf{Year} and set the \textbf{Transfer Type} (\textit{Execute All} or \textit{Execute RT}) from the dropdown.
  \item Check the months you want to assign using the month checkboxes or the \textit{Q1--Q4 / All / None} quick-select buttons.
  \item Click \textbf{+ Add Selected Months} to add them to the \textit{Assigned Periods} table below.
  \item Repeat steps 1--3 to add more months with a different transfer type.
  \item Click \textbf{Execute All} in the header toolbar to run all assigned periods in sequence.
\end{enumerate}
